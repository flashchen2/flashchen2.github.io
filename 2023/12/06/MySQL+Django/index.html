二、django自带ORM框架操作MySQL - 推荐使用

1. 准备工作

  1. ORM是一个翻译的过程，方便操作数据库

  2. 安装第三方模块

  3. ORM可以帮助我们做的两件事

    1. 创建、修改、删除数据库中的表（不用写SQL语句），但是需要手动创建数据库
    2. 操作表中的数据（不用写SQL语句）

2. 创建/连接数据库

  1. 创建数据库

    1. 启动MySQL服务:mysql -u root -p
      Enter password:Pooh_0505
    2. 自带的工具创建数据库：create database 数据库名字 DEFAULT CHARSET utf8 COLLATE utf8_general_ci;
    3. 查看mysql目录下文件夹：show databases;

  2. 连接数据库

    1. 在settings.py文件中进行配置和修改,连接MySQL
    1. 注释掉原有DATABASES数据，重新写入

3. django操作表

1. 创建表 - 级联删除&置空*

  1. app.【models】文件夹书写，确定表的格式
    1. 此时数据库表的名称为数据库的名称

  1. 在pycharm终端解释器执行，余MySQL中生成对应的表

    1. 方法1：

    2. 方法2:

      1. Tools - Run manage.py Task

      2. 在下方终端输入

  2. 运行后会自动创建【settings】已经注册项中的所有表

2. 删除表

  1. app.【models】文件夹注释表格内容

  2. 在pycharm终端解释器重新执行

  3. 【models】注释内容后，再次终端执行，将删除数据库中的内容

3. 修改表

  1. app.【models.py】写入数据库相关内容

  2. 在pycharm终端解释器执行

  3. 【models】中添加新内容后，再次终端执行。在表中新增列时，由于已存在列中可能已有数据，所以新增列必须要指定对应的数据，此时有接下来三个选项

    1. 手动输入一个值
    2. 设定默认值：age = models.IntegerField(default=2)
    3. 允许为空：age = models.IntegerField(null=True, blank=True)

4. 操作后的同步数据库+验证结果

  1. 同步数据库
    在pycharm终端解释器执行，余MySQL中生成对应的表
  1. 验证结果：
    访问mysql：mysql -u root -p
    重置密码：set password = password("Pooh_0505")
    查看mysql目录下文件夹：show databases;
    进入数据库：use 数据库名字；
    查看文件夹下所有的数据表：show tables；
    查看新建表中的数据：desc 数据表名;
    查看新建表中的数据：select * from 表的名称;

四、tips

1. 设置数据库元素允许非空

  1. email字段被定义为非必填字段。
  2. null=True：数据库中该字段可以为空
  3. blank=True：表单中该字段可以为空

2. 建立父、子表单，并将子表单的某值固定设置，为其对应的parent_id

  1. 在子页面get请求，获取子表中，所有含有parent_id的元素
  2. 在子页面post请求，提交表单，将parent_id设为对应父表的nid
